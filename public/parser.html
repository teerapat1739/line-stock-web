🤔 ความท้าทาย: ภาษาไทยไม่เว้นวรรคระหว่างคำ

คนไทยพิมพ์ "เบิกพริก" ติดกันได้ตามปกติ — ไม่เหมือนภาษาอังกฤษที่มีช่องว่างระหว่างคำชัดเจน parser ต้องรู้ว่าจะ "ตัด" ตรงไหน

ที่คนพิมพ์ความหมายที่ต้องการ
เบิกพริก 300 (ติดกัน)เบิก / พริก / 300
เบิก พริก 300 (เว้นวรรค)เบิก / พริก / 300
เบิกไก่ไทย 3 ตัวเบิก / ไก่ไทย / 3 / ตัว
มีน้ำมันพืช 1.5 ลิตรมี / น้ำมันพืช / 1.5 / ลิตร
เบิกพริก300 (ไม่เว้นเลย)เบิก / พริก / 300 (เผื่อใจ)

หัวใจของวิธีนี้: "action" เป็นชุดคำที่รู้ล่วงหน้า" (มี/เติม/เพิ่ม/เบิก/ออก/เกิน) — เลยจับจุดเริ่มได้แน่นอน ส่วน "qty" เป็นตัวเลขท้าย ๆ — จับจุดจบได้ด้วย ที่อยู่ตรงกลาง = ชื่อของ

🪜 อัลกอริทึม 3 ขั้น

1

หา "การกระทำ" จากต้นข้อความ

เทียบกับชุดคำที่รู้: มี / เติม / เพิ่ม / เบิก / ออก / เกิน ถ้าเจอ ตัดออกจากต้น

ตัวอย่าง: เบิก|พริก 300 → action = เบิก

2

หา "จำนวน" + "หน่วย" จากท้ายข้อความ

หาตัวเลข (อาจมีจุดทศนิยม) ที่อยู่ใกล้ท้ายสุด ตามด้วยคำสั้น ๆ ที่อาจเป็นหน่วย

ตัวอย่าง: พริก|300 → qty = 300, unit = ว่าง

3

ที่เหลือตรงกลาง = "ชื่อของ"

หลังตัดหัว (action) และท้าย (qty + unit) ออก สิ่งที่เหลือคือชื่อของ — trim ช่องว่างแล้วใช้ได้

ตัวอย่าง: พริก → item = พริก

ถ้า unit ว่าง → ดูจาก lookup table (ไก่ → "ตัว", ทั่วไป → "กรัม")

🔍 Regex ที่ใช้ — แยกทีละชิ้น

parser ใช้ regex 2 ตัว ตามอัลกอริทึมข้างบน ตัวแรกจับ action ตัวที่สองจับ qty/unit

① จับ action ที่ต้นข้อความ

const ACTION_RE = /^(มี|เติม|เพิ่ม|เบิก|ออก|เกิน)\s*(.*)$/;
^
ต้องเริ่มต้นด้วย — ไม่ใช่ตรงกลาง
(มี|เติม|...)
หนึ่งใน คำกริยาที่กำหนด เท่านั้น ถ้าไม่ตรง = ไม่ใช่ข้อความบันทึก
\s*
ช่องว่างหรือไม่ก็ได้ ("เบิก พริก" และ "เบิกพริก" จะแม็ตช์ทั้งคู่)
(.*)$
ที่เหลือทั้งหมด = ส่วนของ "ของ + จำนวน + หน่วย"

② แยก "ของ + qty + unit" จากที่เหลือ

const TAIL_RE = /^(.+?)\s*(\d+(?:\.\d+)?)\s*([฀-๿a-zA-Z./]*)$/;
(.+?)
"ของ" — จับให้ สั้นที่สุด (non-greedy ด้วย ?) เพื่อให้เลขที่ตามมาเก็บได้ครบ
\s*
ช่องว่างระหว่าง "ของ" กับ "จำนวน" หรือไม่มีก็ได้
(\d+(?:\.\d+)?)
"จำนวน" — รับเลขจำนวนเต็มหรือทศนิยม (เช่น 300 หรือ 1.5)
\s*
ช่องว่างระหว่าง "จำนวน" กับ "หน่วย" หรือไม่มีก็ได้
[฀-๿...]*
"หน่วย" (อาจไม่มี) — รับตัวอักษรไทย (Unicode ฀-๿), อังกฤษ, จุด, slash เช่น "ตัว", "kg", "กก.", "ขวด/ลัง"
$
ต้องจบที่นี่ — ถ้ามีอะไรเกินมา = ข้อความผิดรูป

💻 โค้ดเต็ม — supabase/functions/line-webhook/parser.ts

รันบน Deno (Supabase Edge Functions) — ใช้ได้กับ Node เหมือนกัน ไม่มี dependency

// === Constants ===

type Kind = "in" | "out" | "adj";

// คำกริยาที่รู้จัก → ชนิดของการกระทำ
const ACTIONS: Record<string, Kind> = {
  "มี":   "in",
  "เติม":  "in",
  "เพิ่ม": "in",
  "เบิก":  "out",
  "ออก":  "out",
  "เกิน":  "adj",
};

// หน่วย default ตามชนิดของ (เผื่อ user ไม่พิมพ์หน่วย)
const DEFAULT_UNITS: Record<string, string> = {
  "ไก่ไทย": "ตัว",
  "ไก่":    "ตัว",
  "หมูสับ": "ถุง",
  "น้ำตาล": "ถุง",
  "เกลือ":  "ถุง",
  "ข้าวสาร": "กก.",
};

// === Regex (ดู comment อธิบายในหน้าเว็บ parser.html) ===

const ACTION_RE = /^(มี|เติม|เพิ่ม|เบิก|ออก|เกิน)\s*(.*)$/;
const TAIL_RE   = /^(.+?)\s*(\d+(?:\.\d+)?)\s*([฀-๿a-zA-Z./]*)$/;

// === Public API ===

export interface ParsedRecord {
  action: Kind;
  actionWord: string;   // เก็บคำเดิมไว้สำหรับแสดงผล
  item: string;
  qty: number;
  unit: string;
}

export function parseRecord(raw: string): ParsedRecord | null {
  const text = raw.trim().replace(/\s+/g, " ");
  if (!text) return null;

  // ① หา action จากต้นข้อความ
  const a = text.match(ACTION_RE);
  if (!a) return null;
  const actionWord = a[1];
  const rest = a[2].trim();
  if (!rest) return null;

  // ② แยก ของ + qty + unit
  const t = rest.match(TAIL_RE);
  if (!t) return null;
  const item = t[1].trim();
  const qty  = parseFloat(t[2]);
  const unit = (t[3] || "").trim() || DEFAULT_UNITS[item] || "กรัม";

  // ③ validate
  if (item.length === 0 || !isFinite(qty) || qty <= 0) return null;

  return {
    action: ACTIONS[actionWord],
    actionWord,
    item,
    qty,
    unit,
  };
}

❌ เมื่อ parse ไม่ผ่าน — เกิดอะไรขึ้น

parser คืน null เมื่อข้อความไม่ใช่รูปแบบบันทึก — Edge Function จะส่งต่อให้ขั้นถัดไป:

A

เป็นทักทาย / ขอบคุณ / สวัสดี

ส่งให้ chitchat.ts ตอบสุภาพ — "สวัสดีค่ะ 👋"

B

ใกล้เคียง action แต่สะกดเพี้ยน

คำนวณ Levenshtein distance เทียบกับชุด action ถ้าห่าง 1 ตัว → แนะนำ "ใช่ เบิกพริก 300 หรือเปล่าคะ?"

C

เป็นคำสั่ง (ผิด / ดูสต็อค / วิธีใช้)

match กับ command keyword set — เข้าสู่ flow ตามคำสั่ง

D

ไม่ใช่อะไรเลย

ตอบสุภาพ: "ขอโทษค่ะ ไม่เข้าใจ — ลองพิมพ์เช่น เบิกพริก 300 หรือกดปุ่มข้างล่าง"

🤓 ทำไมถึงไม่ใช้ NLP/LLM

โดเมนของเราจำกัด — action มีแค่ 6 คำ จำนวน = ตัวเลข หน่วย = คำสั้น ๆ ทำให้ regex 2 ตัวเพียงพอ:

  • เร็ว — parse 1 ข้อความ < 1 มิลลิวินาที (Edge Function รวดเร็ว)
  • ฟรี — ไม่ต้องเรียก API ภายนอก ไม่ต้องจ่าย token
  • คาดเดาได้ — ผลลัพธ์เหมือนเดิมเสมอ ไม่ "หลุด" อย่าง LLM
  • Debug ง่าย — input อะไร output อะไร เห็นชัด ทดสอบเขียน unit test ได้ตรง
  • ขยายต่อได้ — เพิ่มคำใหม่ใน ACTIONS หรือ DEFAULT_UNITS ก็พอ

ถ้าในอนาคตอยากให้รับ "พริกสามขีด" (ตัวเลขเป็นคำ) หรือ "ขอเบิกพริกหน่อย 300 กรัม" (ภาษาธรรมชาติ) — ค่อยใส่ LLM เป็น fallback เมื่อ regex fail ก็ได้

parser.ts เป็นชิ้นเล็ก ๆ แต่สำคัญที่สุด — ทุกข้อความเข้ากลุ่ม ต้องผ่านมันก่อนเสมอ
